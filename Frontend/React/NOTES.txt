State Management Hooks (function) : 
We use this hook to manage the state if a variable.

1. useState()
    -- whenever we use the useState() it will return an array with two values inside it. the array looks as shown below.

    sample : 
        useState() ---> return [initValue, functionToChangeTheValue]

    -- We will use array destructing to separate the data and function
        syntax : 
            const [variableName, setVariableName] =  useState(initialValue)

            -- it will store the initValue inside variableName
            -- it wil store the initValue changing function inside setVariableName
    -- It will re-render the entire component whenever there is a change in it's state.

2. useRef()
    -- Also a state management hook but unlike useState it will not return an array to us. Instead we are going to get an object with a key current
    sample : 
        {current : initvalue}
    -- Whenever you want to update the value of a useRef we have to use the below approach

        syntax : 
            const variable = useRef(initValue); //variable going to hold the object with key current which is holding the initValue. 

            variable.current = updatedValue.

    -- useRef is not capable of re-rendering the component. Instead of re-rendering the component what it will do is to update the value and keep a track of it and when the component re-renders then it is going update the state.


    3. useEffect()
        -- is a hook that will perform a certain operation, whenever we are rendering a component and if that component is having useEffect hook then it will perform the given task.

        render component --> useEffect 

        -- We can restrict when we want the useEffect to work. 

        syntax : 
            useEffect(callbackFun, [dependencyArray,....])
        
        3 types : 

        1. useEffect(callbackFun)
            -- will always execute the callBackFunction when ever we are rendering the component.
        
        2. useEffect(callbackFun, [])
            -- will execute only at the initial rendering. 
        
        3. useEffect(callBackFun, [variable])
            -- This useEffect will execute only when the given variable state changes.

                function ContextComponent({children}) {
                    return (
                        <contextAPI.provider>
                            {children}
                        </contextAPI>
                    )
                }

Routing : 
    -- moving from one page to another page. 

    ex : 
        Mumbai to Hyderabad -- Route

        Mumbai -- Banglore -- Hyderabad -- Route1 
        Mumbai -- Maharastra -- Hyderabad -- Route2 
        Mumbai -- Maharastra -- Karanataka -- Hyderabad -- Route3

    -- From one page --> another 
        -- Define the route to the pages. 
        -- But react is not capable of routing the pages.
        -- To make react work with the routing -- React router 
            -- In react router we will get
                -- Router -- will keep the routes 
                    Syntax : 
                        <Router>
                            <Routes>
                                <Route></Route>
                            </Routes>
                        </Router>
        -- We access or navigate to another page with the help of 'endpoint'
            -- https://localhost:5036/(Base url)

            Adding endpoint
            -- https://localhost:5036/EndPoint
                ex : 
                    -- https://localhost:5036/contact-us (url with EndPoint)
                                (or)
                    -- https://localhost:5036/user/1 (url with EndPoint + Parameter)
        -- We have to define the Route/path for each page. 
            -- Route contains two important parts.
                1. path -- here we define the endpoint 
                2. element -- this will be the rendering element when user hits the endpoint.
                    syntax : 
                        <Route path='/endpoint' element={<ComponentName />}></Route>
                                        (or)
                        <Route path='/endpoint/:paraName' element={<ComponentName />}></Route>

                    Example :
                        <Route path='/contact-us' element={<Contact />}></Route>
                                        (or)
                        <Route path='/user/:user_id' element={<UserProfile />}></Route>

        -- Link -- link is a anchor tag when it is converted to html element. 
        -- in react instead of using anchor tag we will use Link Component to navigate from one page to another.
            -- syntax ; 
                <Link to='/endpoint'>text</Link>
                    (or)
                <NavLink to='/endpoint'>text</NavLink>
        -- NavLink -- help you to navigate from one page to another but the active component will get a class-name active.
        -- we have to wrap the Links and Routes inside the Router.

Redux Tool kit :

    -- We will divide the context api data into slices and each slice will contain it's own data. 
    -- redux is going to combine all those slices into a single data or object with the help of store.
    -- To update the values inside a slice we have to use reducers(action).
    -- The values are called as state. The state can only updated with the help of the reducers.
    -- We will wrap the components/elements which will share the data among them with the Provider component.
    -- To the provider component we need to use the store -- component.
    -- Redux slices can't handle the async code. --> use asyncThunk 
    -- asyncThunk is a hook which will be used in redux toolkit to fetch the data.
    -- we will create extraReducers to handle the asyncthunk 
        -- pending -- data is being fetched
        -- fulfilled -- data was successfully fetched 
        -- rejected -- data was failed to fetch.
